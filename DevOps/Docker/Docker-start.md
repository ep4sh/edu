# Docker start
  
#### 0. Для ориентации:
   * *__Docker Image__* (докер имидж) - легковесный, обособленный кусок твоего говно-кода и таких же библиотек, готовый к запуску в космос или куда по-дальше, управляется Dockerfil`ами.
   * *__Docker Container__* - это запущенный инстанс твоего имиджа
   * *__Docker Service__* - часть распределнного приложения, управляются .yaml-файлами

#### 1. Установить Docker

#### 2. Создать директорию, в ней написать *Dockerfile*

#### 3. Создать Docker-Имидж:
```docker build -t <imgName> .```

`#вывести список всех имиджев: `  
```docker images ```

##### 4. Запуск контейнеа, ключ -p мапит локальный порт 7000 на 80 порт контейнера:
```docker run -p 7000:80 <imgName>```

##### 5. Некоторые полезные фичи docker run:
   * -d #детач - запуск в бэкграунде и вывод Сid
   * -dns # установить DNS
   * -h # hostname
   * -i # интрекативный - сохраняет STDIN, даже если контейнер не аттачен
   * -m # лимит памяти
   * --mac-address # САБЖ
   * --ip # тоже сабж
